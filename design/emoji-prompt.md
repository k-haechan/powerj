# powerj Custom 3D Icon Set — Production Prompt v3.0 (10/10)

> 이미지 생성 모델(GPT Image, Midjourney, Imagen, Flux 등)에 그대로 투입하는 마스터 스펙.
> 시스템 이모지를 대체하는 **완벽하게 통일된 앱 아이콘 18종**을 만든다.
> 우선순위는 **창의성이 아니라 통일성** — 18종 모두 같은 산업 디자이너가 같은 앱을 위해 그린 것처럼 보여야 한다.
> 기준 문서: `design/DESIGN.md` v2 (마켓 스탠다드 라이트).

## 사용 환경 (왜 이 스펙인가)

- 아이콘은 **38×38px 라운드 사각(12px) 틴트 배지** 안에 놓인다 (`components.css .ico`). 작게 렌더되므로 실루엣이 강해야 한다.
- 배경은 웜 오프화이트(`#F6F4EF`) 위의 흰 카드, 배지 틴트는 파스텔(인벤토리 표 참조).
- 탭바는 SVG 라인 아이콘으로 이미 교체됨 — 이 세트는 **카테고리 배지·모드 카드·패널 뱃지 전용**.
- 컨셉: 매트한 세라믹/클레이 질감의 미니 3D 산업 오브제. 얼굴·눈 금지(마스코트 없는 제품), 장난감이 아니라 도구.

## MASTER PROMPT (템플릿)

`{SUBJECT}` 와 `{MAIN_COLOR}` 두 값만 교체한다. 나머지는 **바이트 단위로 동일**하게 유지 — 이것이 세트 통일성의 핵심이다.

```
A single {SUBJECT}, rendered as a premium soft clay-like matte plastic object with a very fine ceramic texture and subtle smooth gradients. Clean industrial product design, premium mobile application asset, NOT handmade clay.
Minimal geometric form with fully rounded corners, smooth continuous surfaces, no sharp edges, no unnecessary details.
Orthographic front view, tilted downward by approximately 12 degrees. Very little perspective distortion. Identical camera angle as every other icon in the set.
One soft area light from the upper left at approximately 45 degrees. Gentle self-shading only. No ambient occlusion outside the object. NO ground shadow. NO floating shadow. NO floor. NO vignette.
Use one dominant color: {MAIN_COLOR}. Use at most one neutral accent (#F6F4EF, #FFFFFF or #211B12) and optionally one secondary accent chosen ONLY from this exact palette: #F07A2B, #FF9D4D, #E9B93B, #7189A8, #5B9BD5, #3FB58F, #EF7C66, #9B7EDE, #E77FB3. Do not introduce any other colors.
Strong, bold, recognizable silhouette. Readable at 38x38 pixels. Avoid thin parts, excessive detail, tiny cutouts or fragile geometry.
Center the object precisely. Occupy approximately 80% of the canvas with equal padding on all four sides.
Transparent alpha background only. No background. No canvas. No environment. No backdrop.
Every icon must appear as if designed by the same industrial designer for the same productivity application. The entire collection must share identical camera angle, lighting, proportions, material, rendering quality, palette and visual language. Consistency is far more important than creativity.
Output as a premium production-ready mobile app icon. Not an illustration. Not concept art. Not a mascot. Not a toy. Not a collectible. Not a photo.
NO face. NO eyes. NO mouth. NO expressions. NO characters. NO text. NO letters. NO numbers. NO logos. NO symbols. NO outline stroke. NO sticker border. NO glossy highlights. NO metallic reflections. NO glass. NO photorealism. NO fabric texture. NO wood texture. NO rough handmade clay. NO emoji style. NO Apple emoji. NO Google emoji. NO Microsoft emoji.
```

## GENERATION WORKFLOW

**STEP 1 — 앵커 확정.** `ico-commute`(#2)를 먼저 생성한다. 4변형 요청 → 가장 좋은 1개 선택. 이것이 세트의 시각 앵커.

**STEP 2 — 나머지 17종.** 이미지 모델이 스타일 레퍼런스를 지원하면 앵커 이미지를 물려서 생성한다 (GPT Image → reference image / Midjourney → `--sref` / Flux → reference image). **세트 전체를 각자 독립 생성하지 말 것.** 미지원 모델이면 위 완성 프롬프트 문구만으로 진행.

**STEP 3 — 18개 동시 검수.** 한 장에 18개를 나란히 놓고 아래가 전부 동일한지 본다:
✓ 카메라 앵글 ✓ 조명 ✓ 재질 ✓ 렌더 품질 ✓ 비례 ✓ 스케일 ✓ 패딩 ✓ 부드러움 ✓ 색 채도 ✓ 시각 무게. **어느 하나도 스타일적으로 튀면 안 된다.**

## 후처리 & 앱 적용

1. 배경 투명 확인 → 1024 원본 보관 → 앱 투입용 `76px`(2x)·`114px`(3x) PNG 익스포트 → `design/system/assets/icons/ico-*.png`
2. `components.css .ico` 의 이모지 텍스트를 `<img src=".../ico-*.png" width="22" height="22" alt="">` 로 교체. 배지 틴트 클래스(sky/mint/coral/violet/rose/ember/slate)는 인벤토리 표 매핑 유지.
3. 아이콘 자체가 색을 갖게 되면 배지 틴트는 낮은 채도의 배경으로만 작동 — 아이콘 주 색과 틴트가 같은 계열로 뭉개지지 않게 대비 확인.
