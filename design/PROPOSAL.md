# powerj 디자인 시스템 — 최종 제안 (검토용 · 아카이브)

> ✅ **결정 완료 (2026-07-04)**: 5개 시안(`design/sample.html`) 검토 후 **B안 엠버 크로노그래프** 확정.
> 정식 스펙은 `design/DESIGN.md` 참조. 이 문서는 검토 과정 기록용 아카이브다.

> `/design-consultation` 리서치 + 독립 디자인 보이스 종합 결과.
> 이 문서는 **검토용 제안서**입니다. 방향이 확정되면 `design/DESIGN.md`(단일 진실 원천)로 정식 작성됩니다.

## 전제

- **제품**: 극한 계획/실행 앱. 프로세스 체인 + 정량 목표 + 타이머/방해금지 + 전환 검증 + 일기/대시보드 마감.
- **플랫폼**: 모바일 우선.
- **기억될 한 가지**: 매일 쌓이는 성취의 쾌감 — 하루 마감 대시보드가 보상의 순간.
- **원칙**: 실패는 벌이 아닌 데이터.

## 리서치 요약 (browse 캡처 5종 + 웹서치)

| 제품 | 시각 언어 | 감정 포지션 |
|---|---|---|
| Structured | 코랄 핑크, 라운드, 일러스트 | 귀여운 격려 |
| Sunsama | 크림+오렌지, 여백, 차분함 | 마음챙김 리추얼 |
| Rize | 다크+보라, 데이터 밀도 | 차가운 측정 |
| Forest | 딥그린, 새싹 게이미피케이션 | 귀여운 보상 |
| Opal | 흑색 럭셔리, 대형 수치, 마일스톤 젬 | 다크 럭셔리 측정 |

**수렴점(table stakes)**: 세로 하루 타임라인, 큰 숫자 통계, 진행 링/바, 타이머 전면 배치.

**EUREKA**: 카테고리는 "측정=차갑게, 보상=귀엽게"로 갈라져 있다. powerj는 **측정 자체가 보상**인 제품 — 이 빈 자리가 우리의 자리다.

## A안 (추천) — 활판 텔레메트리 (Letterpress Telemetry)

**비주얼 테제**: 하루를 활자로 조판하고, 인쇄기에 걸고, 도장으로 봉인한다. 성취는 종이에 내려앉는 잉크다.

- **무드**: 아침 7시의 인쇄 공방. 진지한 따뜻함. 귀엽지도, 감시하지도 않는 "장인의 작업장".
- **한국적 근거**: 도장/인주 문화, "참 잘했어요" 도장을 성인용·정량적으로 재해석. 경쟁사가 건드릴 수 없는 정서.

### 컬러 (페이퍼 라이트 우선)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--paper` | `#F3EEE4` | 배경 — 따뜻한 비도공지 |
| `--slip` | `#FBF8F0` | 서피스 — 작업 전표/카드 |
| `--rule` | `#D8D0BE` | 헤어라인 괘선 (그림자 대신) |
| `--ink` | `#221F1A` | 본문 잉크 (대비 ≈13:1) |
| `--ink-muted` | `#6F675A` | 보조 텍스트 |
| `--vermilion` | `#E0491F` | **인주 — 성공 전용.** 도장, 100% 봉인 |
| `--press-blue` | `#2D4FC8` | 계획·데이터·실패 원인 (편집자의 볼펜) |
| `--reclaim-green` | `#00845C` | 시간 단축 (절제해서) |
| `--burnt` | `#8C2F14` | 파괴적 액션 전용 (삭제 등) |

- **저녁판**: 일몰 후 자동 반전 — paper `#211D18`, ink `#EFE7D3`, vermilion `#FF6A3D`, blue `#7D96FF`. "다크모드"가 아니라 반전 인쇄판.
- **실패에 빨강 없음.** 실패 데이터는 볼펜 파랑 — 읽는 것이지 느끼는 것이 아니다.

### 타이포그래피 (전부 무료·즉시 사용 가능)

| 역할 | 폰트 | 이유 |
|---|---|---|
| 디스플레이 한글 | **Paperlogy** | 포스터 에너지, 템플릿 냄새 없음 (업그레이드 옵션: 단조) |
| UI 본문 한글 | **Wanted Sans Variable** | 작은 크기 렌더링 우수, 중립적 |
| 일기(사람의 목소리) | **MaruBuri** | 기계의 숫자 vs 사람의 세리프 — 일기 화면만 세리프 전환 |
| 타이머 히어로 숫자 | **Doto** (가변) | 도트 매트릭스 = 프린터 헤드. 달성률이 오를수록 도트가 진해짐 |
| 데이터/원장 숫자 | **Martian Mono** | tabular figures, 계기판 모노 |

### 레이아웃

- 좌측 원장(ledger) 마진 고정, 4pt 베이스라인 그리드
- 그림자 대신 헤어라인 괘선, 라운딩 0–2px
- 시그니처 형태: **절취선 노치** (라운드 사각형이 아님)
- 중앙 정렬은 "도장 찍는 순간"에만

**히어로 화면 3종:**
1. **실행 타이머** — 뷰포트 60%를 채우는 거대 Doto 숫자(우측으로 블리드), 프로세스명은 우측 마진에 세로쓰기. 목표 진행은 `영상 2/2 · p.147/200` 조판 라인.
2. **전환 검증 게이트** — 끝난 프로세스가 영수증처럼 위로 찢어짐 → 달성률 엄지 래칫 다이얼(10%마다 햅틱 클릭, 클릭마다 잉크 밴드) → 실패 원인은 활자 타일 한 탭 → **꾹 눌러 도장 쾅**(매번 미세하게 다른 잉크 자국). 4초 안에 완료.
3. **하루 마감 대시보드** — 도트 프린터처럼 위→아래로 인쇄되는 하나의 교정쇄(~1.8s). 날짜는 소인(postmark), 총 달성률은 포스터 스케일, 시간 원장(단축=초록/초과=파랑), 실패 원인은 펀치카드 도트 그리드, 일기는 종이가 한 톤 따뜻해지며 MaruBuri로. 과거 = **전표 더미의 두께가 곧 성장 그래프.**

### 모션·보상 언어

- 진행 프리미티브는 오직 **잉크 차오름** (슬라이드 없음, 포화됨)
- 도장 쾅: 무거운 햅틱 + 80ms 스쿼시 — "좋은 차 문 닫는 느낌"
- 사운드 3개만(찢기/래칫/도장), 실녹음, 토글 하나로 전부 끔
- 콘페티·파티클·마스코트·스트릭·XP 없음. **더미가 곧 스트릭.**
- reduced-motion: 인쇄→페이드, 찢기→컷, 햅틱은 유지

### SAFE / RISK

**SAFE**: 세로 타임라인, 큰 수치+진행 표시, 기능적 마이크로 인터랙션.

**RISK**:
1. **페이퍼 라이트** — 다크가 "진지함"인 카테고리에서 종이로 승부. 얻음: 즉각 차별화, 실패 기록의 탈드라마화. 잃음: OLED 배터리 스토리, 스크린샷의 "테크함".
2. **주홍=성공** — 빨강을 처벌→승인(도장)으로 반전. 얻음: "실패는 데이터"의 지각적 구현. 잃음: 색 관습과의 싸움(형태 이중화로 보완: 성공=도장 형태, 실패=펀치홀 형태).
3. **게이미피케이션 제거** — 성인용 진지함 확보, 검증된 후크 포기.

### 첫 3초의 목표 반응

오늘의 빈 전표 + 흐린 `100%` 워터마크 + "오늘 몇 %를 인쇄할까요?" → **"오늘 저 도장 받고 싶다."**

## B안 (대안) — 엠버 크로노그래프 (Ember Chronograph)

**비주얼 테제**: 다크 웜 차콜 계기판 위에서 달성률이 엠버(불씨)빛으로 차오른다.

- **컬러**: 웜 차콜 `#141210` 배경 / 엠버 `#E8853D` 계열이 달성 heat / 실패 데이터는 쿨 슬레이트 / 100%=골드.
- **타이포**: 숫자 **IBM Plex Mono**, 본문 **Wanted Sans**.
- **평가**: 타이머·집중 모드와 잘 어울리고 카테고리 문법에 가까워 안전. 그러나 Opal(흑색 럭셔리)·Rize(다크 데이터)의 그림자 안 — "기억될 한 가지"를 만들기엔 A안 대비 약함.
